# 本地部署与运行指南

## 1. 代码在哪里

仓库采用 monorepo 结构，核心代码位置如下：

- `apps/api/`：后端 API 聚合层，入口为 `apps/api/app/main.py`
- `apps/web/`：前端管理台，采用 Next.js + TypeScript
- `packages/agent-runtime/`：Agent 运行时与内置 Agent 模板
- `packages/model-gateway/`：模型 provider 抽象层
- `packages/tool-hub/`：工具注册与执行中心
- `packages/knowledge-service/`：知识库服务
- `packages/memory-service/`：短期/长期记忆服务
- `packages/audit-service/`：审计日志服务
- `packages/shared-schema/`：平台公共 schema
- `packages/im-adapter/`：IM adapter 抽象与 mock 实现
- `infra/scripts/` 与 `scripts/`：本地开发脚本
- `docs/`：PRD、架构和部署文档

## 2. 运行前准备

### Python

建议使用 Python 3.10+。

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements-dev.txt
```

### Node.js

建议使用 Node.js 20+。

```bash
npm install
```

## 3. 方式一：分别启动后端和前端

### 3.1 启动后端 API

在仓库根目录执行：

```bash
make api
```

启动后默认监听：

- API: `http://localhost:8000`
- 健康检查: `http://localhost:8000/health`

你也可以直接执行：

```bash
PYTHONPATH=apps/api:packages/shared-schema:packages/agent-runtime:packages/model-gateway:packages/tool-hub:packages/knowledge-service:packages/memory-service:packages/audit-service:packages/im-adapter python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000 --app-dir apps/api
```

### 3.2 启动前端管理台

在另一个终端执行：

```bash
make web
```

启动后默认访问：

- Web: `http://localhost:3000`

## 4. 方式二：一键开发启动

```bash
bash scripts/dev.sh
```

或：

```bash
bash infra/scripts/dev.sh
```

这个脚本会同时拉起后端和前端开发服务。

## 5. Docker Compose 方式

在仓库根目录执行：

```bash
docker compose up --build
```

默认端口：

- API: `8000`
- Web: `3000`

## 6. Demo 数据与调试建议

### 初始化说明

当前 MVP 的 demo 数据是代码内置的，不需要额外导入数据库；可执行下面命令查看提示：

```bash
make bootstrap-demo
```

### 推荐调试路径

1. 打开 `http://localhost:3000`
2. 在 Dashboard 查看 3 个内置 Agent
3. 进入 Session 调试页，使用下面示例消息：
   - 制度助手：`请告诉我员工转正申请需要提前多久发起？`
   - 招聘群助手：`帮我查一下候选人和后端岗位JD`
   - 报表助手：`请总结本周招聘报表并生成PPT大纲`
4. 如果只验证 API，可直接调用：

```bash
curl -X POST http://localhost:8000/sessions/debug \
  -H 'Content-Type: application/json' \
  -d '{
    "session_id": "demo-session-001",
    "agent_id": "policy-knowledge",
    "user_id": "demo-user",
    "message": "请告诉我员工转正申请需要提前多久发起？"
  }'
```

## 8. 常见问题

### Q1：为什么我只想跑后端？

直接执行：

```bash
make api
```

### Q2：为什么前端打不开数据？

请确认：

1. 后端已启动在 `http://localhost:8000`
2. `.env.example` 中的 `NEXT_PUBLIC_API_BASE_URL` 与实际地址一致
3. 前端已执行 `npm install`

### Q3：这个版本是不是完整生产版？

不是。

当前是 MVP 演示骨架，重点是：

- 本地可运行
- Agent 配置与调试链路可演示
- 便于后续替换真实模型、向量库、数据库、IM adapter

部分能力仍为 stub / mock：

- 文档检索是 demo 检索逻辑
- 工具为 mock 返回
- artifact 仅创建任务，不真正生成文件
- IM adapter 仅提供 mock 实现
